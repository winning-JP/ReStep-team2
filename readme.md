# 健康促進のアプリ
## アプリ名: ReStep (Reward Step)
## アプリ概要（何をするアプリか）
- 目的：ユーザーが努力感なく、気づいたら健康的な行動を取れている状態を作る
- 価値：健康知識を教えるより、行動変容（続く仕組み）を設計する
- 方向性：
- “運動/食事を頑張らせる”ではなく、日常の流れの中で自然に行動が変わる導線を作る
- AI前提で、AIだけで完結しない体験（リアルとの接点や環境設計）を持たせると良い

## 想定ターゲット層（誰向けか）
### メインターゲット（最重要）
- 運動が苦手・嫌い・面倒で続かない人
- 健康意識はある/ないどちらでもOKだが、共通点は「続かない」

### 具体例
- 20〜40代：仕事で疲れていて運動の優先度が低い人
- 学生：生活リズムが乱れがちで、健康行動が習慣化しない人
- 家族持ち：自分の時間が取れず運動が途切れる人

**`※重要：チョコザップ契約者向けではない／一般の人（自分や友達・家族が使う想定）`**

## 提供する体験（どうやって健康にするか）
- キーコンセプト：努力ゼロ化 / 習慣の自動化
- 例として入れやすい要素
- 小さすぎる行動（30秒〜2分）を起点にする
- “やる気”ではなく トリガー（時間・場所・行動）で動かす
- できたら褒めるだけでなく、次の行動が勝手に発生する設計（環境・予約・準備など）

## AIの使い方（前提条件）
- AIは使ってOK。
  - ただし「AIが提案します」だけだと弱い
- リアル行動に接続（通知、音声、位置、デバイス、生活導線、友人との約束など）して“動かす”ところが肝

## NGになりやすい方向（避けたい）
- チョコザップ前提・店舗機能前提の設計（基本は不要）
- オンライン完結の「指先だけ」サービス（できれば避ける）
- “健康情報まとめ/アドバイス”だけのアプリ（行動が変わらない）

## 入れたい機能
- 回る機能
- 歩数
	- 消費カロリー
- 位置情報
- ご褒美機能
	- ゲーム感覚 (スタンプラリーなどデジタルコンテンツの特典)
- 食事管理
- 睡眠
- ストレス検知
- 心電図
- 体重計と連携
- バックグラウンドで記録
	- アプリを開かなくても歩数などを記録 → 	HealthKitを定期的に取得?
- AIコーチング機能 (AIが指図する)
- 目標機能

# 絶対に入れたい機能

## APIエンドポイント切替
- 通常は `ReStep/Info.plist` の `API_BASE_URL` を変更してください。
  - 例: `https://staging.example.com`
- 各APIの最終URLは以下に自動展開されます。
  - `user_api.php`
  - `user_profile_api.php`
  - `user_stats_api.php`
  - `user_wallet_api.php`
- 個別に上書きしたい場合のみ `API_ENDPOINT` / `PROFILE_API_ENDPOINT` / `STATS_API_ENDPOINT` / `WALLET_API_ENDPOINT` を指定します。
